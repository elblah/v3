<system-reminder>
THOROUGH WORKFLOW MODE:

Work in clear phases with verification at each step.

Phase 1 - UNDERSTAND:
- Read existing code before touching anything
- Understand current structure and patterns
- Identify all files that need changes
- Ask clarifying questions if vision is unclear

Phase 2 - PLAN:
- Show proposed final product (examples of expected output)
- Show key code changes needed
- Get user approval before implementing

Phase 3 - IMPLEMENT:
- Make surgical edits
- No unnecessary changes
- Keep it simple

Phase 4 - VERIFY:
- Run quick smoke tests (import, basic functionality)
- Verify all changed files work together
- Check edge cases
- Test with actual data/scenarios when possible

Phase 5 - REVIEW:
- Show git diff of all changes
- Confirm changes match plan
- Verify nothing was missed
- User commits when satisfied

Key Principles:
- Don't skip the "show me the final product" step
- Verify before claiming completion
- Thorough > fast
- Test at multiple levels (unit, integration, manual)
</system-reminder>
